⚙️ Stack Tecnológico
Tecnologías seleccionadas para el MVP. Criterio principal: velocidad de desarrollo + ecosistema argentino de pagos.
📱 Mobile
⚛️
React Native
con Expo
✓ Elegido
Un solo código fuente para iOS y Android. Expo acelera el setup y evita problemas nativos en la fase inicial. Enorme pool de devs en Argentina.
Alternativa descartada: Flutter — menor disponibilidad de devs locales, mayor curva de aprendizaje.
🔧 Backend & DB
⚡
Supabase
PostgreSQL + Auth + Storage
✓ Elegido
Todo en uno: base de datos PostgreSQL, autenticación, almacenamiento de fotos y APIs REST/GraphQL generadas automáticamente. Reduce a la mitad el tiempo de backend.
Alternativa descartada: Firebase — NoSQL limita consultas geoespaciales de canchas.
💳 Pagos
💙
MercadoPago
SDK React Native
✓ Elegido
Dominante en Argentina (80%+ market share). Soporta tarjetas, débito, transferencias y pagos en efectivo (Rapipago/Pago Fácil). SDK bien documentado.
Complemento: Transferencia bancaria como opción manual fallback.
🗺️ Mapas & Geolocalización
🗺️
Google Maps
react-native-maps
✓ Elegido
Indispensable para mostrar canchas cercanas. Google Places API para autocompletado de direcciones. Places Nearby para búsqueda por radio.
Costo: ~$0 en volumen MVP (límite gratuito generoso).
🔔 Push Notifications
📲
Expo Notifications
+ OneSignal fallback
✓ Elegido
Integrado directamente en Expo. Cero configuración adicional de APNs/FCM en la fase inicial. Permite notificaciones para confirmaciones, recordatorios y cancelaciones.
OneSignal para analytics avanzados en V2.
☁️ Hosting & DevOps
▲
Vercel + Supabase Cloud
Free tier → Pro en lanzamiento
✓ Elegido
Vercel para el panel web de admins. Supabase Cloud para la DB y APIs. GitHub Actions para CI/CD. Costo inicial ~$0 durante desarrollo.
App stores: Expo EAS Build para generar los .ipa y .apk.
📊 Decisión clave: React Native vs Flutter
Por qué React Native es la elección correcta para este MVP con deadline de 47 días
Criterio ⚛️ React Native + Expo 🎯 Flutter Veredicto
Velocidad de arranque ⚡ Expo → app corriendo en 1 día Setup más complejo, SDK pesado RN ✓
Pool de devs en Argentina 🏆 Enorme, JS/TS es ubicuo Dart tiene menos devs locales RN ✓
Integración con MercadoPago ✅ SDK oficial React Native SDK oficial Flutter (también OK) Empate
Librería de mapas ✅ react-native-maps (madura) google_maps_flutter (OK) Empate
Performance UI Buena (con RN New Architecture) 🏆 Excelente, renderizado propio Flutter ✓
Tiempo al MVP ⚡ 3–4 semanas factible 4–6 semanas estimado RN ✓
Ecosistema de librerías 🏆 npm: miles de paquetes pub.dev: creciendo pero menor RN ✓
VEREDICTO FINAL ✅ ELEGIDO Descartado para MVP React Native

🔄 Flujo de datos — Reserva exitosa
El camino completo de una reserva, de usuario a cancha
1
Usuario busca cancha
App envía lat/lng + radio + filtros → Supabase query con PostGIS (búsqueda geoespacial) → retorna canchas ordenadas por distancia
2
Selecciona horario
Query a reservas table filtrando por cancha + fecha para mostrar slots disponibles. Lock optimista para evitar doble reserva.
3
Pago con MercadoPago
App llama a Supabase Edge Function → función crea preferencia en MP → retorna init_point → app abre checkout de MP → usuario paga
4
Webhook de confirmación
MercadoPago envía webhook a Supabase Edge Function → se actualiza reservas.estado = 'confirmada' → Realtime notifica a la app en tiempo real
5
Notificaciones
Push notification al jugador (confirmación) + al dueño de cancha (nueva reserva) + email transaccional vía SendGrid con los detalles

⚡ Índices y optimizaciones clave
CREATE INDEX canchas_ubicacion_gix ON canchas USING GIST(ubicacion); -- búsqueda por radio
CREATE INDEX reservas_cancha_fecha_idx ON reservas(cancha_id, fecha); -- consulta de disponibilidad
CREATE UNIQUE INDEX una_review_por_reserva ON reviews(reserva_id); -- 1 sola reseña
ALTER TABLE reservas ADD CONSTRAINT no_double_booking EXCLUDE USING gist(cancha_id WITH =, tsrange(fecha+hora_inicio, fecha+hora_fin) WITH &&); -- doble reserva
📱 Flujos UX/UI Principales
Los 4 flujos críticos del MVP — cada uno es un camino de usuario de punta a punta
A Registro & Onboarding
1
Splash + elección de rol
El usuario elige: "Soy jugador" o "Tengo una cancha". Define el flujo completo.
SplashScreenRolSelectionScreen
2
Autenticación
Registro con email+contraseña o Google OAuth. Verificación de email automática vía Supabase Auth.
AuthScreen
3
Completar perfil
Nombre, teléfono, foto opcional. Si es dueño: nombre de la cancha, dirección y fotos.
ProfileSetupScreen
4
Permisos y Home
Solicitar permiso de localización + notificaciones. Redirige a HomeScreen (jugador) o DashboardScreen (dueño).
PermissionsScreenHomeScreen
B Búsqueda & Reserva
1
Búsqueda con mapa
Mapa centrado en ubicación del usuario. Pines de canchas disponibles. Filtros: fecha, hora, tipo de cancha, precio.
SearchScreenMapView
2
Detalle de cancha
Fotos en carrusel, precio, amenities, reseñas, calificación promedio, dirección con Google Maps.
CanchaDetailScreen
3
Selección de horario
Calendario con slots disponibles en verde y ocupados en gris. El usuario elige fecha y hora.
BookingScreen
4
Confirmación
Resumen de la reserva: cancha, fecha, hora, precio. Botón "Confirmar y pagar".
BookingConfirmScreen
C Flujo de Pago
1
Elección de método de pago
3 opciones: MercadoPago (tarjeta/QR/cuotas), Transferencia bancaria, Efectivo en cancha.
PaymentMethodScreen
2
Checkout de MercadoPago
Se abre el SDK nativo de MP con el precio ya cargado. El usuario completa el pago en la interfaz de MP.
MPCheckout (externo)
3
Retorno a la app
Deep link regresa a FulboApp. Se muestra loading mientras el webhook de MP confirma el pago.
PaymentLoadingScreen
4
Reserva confirmada 🎉
Pantalla de éxito con los detalles. Push notification al jugador y al dueño. Email automático.
BookingSuccessScreen
D Panel Admin — Dueño de Cancha
1
Dashboard principal
Resumen del día: reservas de hoy, ingresos del mes, calificación promedio, nuevas reseñas.
AdminDashboardScreen
2
Gestión de agenda
Vista semanal con reservas confirmadas, pendientes y slots libres. Puede bloquear horarios manualmente.
AdminCalendarScreen
3
Gestión de la cancha
Editar fotos, precios, horarios disponibles, amenities. Activar/desactivar la cancha.
AdminCanchaEditScreen
4
Ingresos y retiros
Historial de pagos recibidos, retiros a CBU/CVU. Reporte mensual descargable.
AdminIncomesScreen
